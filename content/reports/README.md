# 活動報告の投稿方法

このフォルダ（`content/reports/`）にMarkdownファイルを追加するだけで、新しい活動報告を投稿できます。

## ファイルの作成方法

1. 新しいMarkdownファイルを作成します
   - ファイル名: `YYYY-MM-DD-slug.md` の形式（例: `2025-06-15-forum-presentation.md`）
   - `slug` はURLに使われるので、半角英数字とハイフンで記述してください

2. ファイルの先頭にフロントマター（メタデータ）を記述します

```markdown
---
id: "4"
slug: "2025-06-15-forum-presentation"
title: "◯◯フォーラム2025にて研究成果を発表しました"
publishedAt: "2025-06-15T10:00:00+09:00"
category: "研究発表"
tags: ["学会発表", "フォーラム"]
summary: "◯◯フォーラム2025にて、本研究会のプロジェクトに関する研究成果を発表しました。"
---
```

3. その下に本文をMarkdown形式で記述します

```markdown
2025年6月15日、東京都内で開催された「◯◯フォーラム2025」にて、本研究会の学生3名が研究成果を発表しました。

## 発表のポイント

- ポイント1
- ポイント2
- ポイント3
```

## フロントマターの項目説明

- `id`: 一意のID（文字列）
- `slug`: URLに使われる識別子（半角英数字とハイフンのみ）
- `title`: 記事のタイトル
- `publishedAt`: 公開日時（ISO 8601形式: `YYYY-MM-DDTHH:mm:ss+09:00`）
- `category`: カテゴリ（例: "研究発表", "イベント", "メディア掲載"）
- `tags`: タグ（配列形式）
- `summary`: 記事の要約（一覧ページに表示される）
- `thumbnailUrl`: サムネイル画像のURL（任意）

## 例

既存のファイル（`2025-forum-presentation.md`など）を参考にしてください。

## 注意事項

- ファイルを追加・編集した後、開発サーバーを再起動する必要があります
- `slug`は既存のものと重複しないようにしてください
- Markdownの記法（`##` で見出し、`-` でリストなど）が使えます
